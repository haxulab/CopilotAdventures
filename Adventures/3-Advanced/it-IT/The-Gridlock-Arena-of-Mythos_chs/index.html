Mythos的Gridlock竞技场

背景

在神秘的Mythos大陆，各种生物聚集在Gridlock竞技场，一个位于棋盘上的网格中，测试策略、力量和智慧。每个生物都有独特的移动、力量和策略。

目标

你的任务是模拟在Gridlock竞技场中的战斗。每个生物将进行一系列的移动，每次移动后，如果生物落在同一个格子上，则会对对方造成伤害。目标是在战斗结束时积累最高分。为了跟踪战斗进度，每次移动后显示网格，并在网格下显示当前得分。

规范

  1. 网格动态：

  2. Gridlock竞技场是一个5x5的网格。

  3. 网格中的每个格子可以是空的或被生物占据。
  4. 生物可以向上、向下、向左或向右移动一个格子。
  5. 生物数据：
名字 起始点 移动 力量 图标
龙 2,2 右, 左, 下 7 🐉
哥布林 2,3 左, 右, 上 3 👺
兽人 0,0 右, 下, 下 5 👹
3. 战斗动态：
  • 生物轮流移动。
  • 如果两个生物在移动后落在同一个格子上，双方都会对彼此造成伤害。
  • 分数根据造成的伤害分配。
  • 当所有移动完成时，战斗结束。

  • 输出：

  • 每次移动后，通过在控制台中打印网格来显示网格，用⬜️表示空格子。

  • 在网格上方添加一个标题，显示“初始状态”（用于显示网格的初始状态）或“移动 X”，其中X是当前移动的编号。
  • 使用每个生物的图标在网格上表示它们。
  • 空格子用⬜️表示。
  • 战斗格子用🤺表示。
  • 每次移动后在网格下方显示每个生物的当前分数。

  • 在战斗结束时，返回每个生物累计的总分。

约束

  • 使用GitHub Copilot，并用你喜欢的任何语言编写模拟。
  • 确保算法高效，能处理战斗动态。在GitHub Copilot/Chat中询问：“如何使这个代码更具可读性和可维护性？”。
  • 程序应具有100%的测试覆盖率。使用GitHub Copilot Chat中的/tests命令。

高层次任务摘要

  1. 使用控制台应用程序进行输出。

  2. 定义常量和数据结构：

  3. 定义包含生物详细信息的creatures数组。

  4. 定义一个directions对象，用于将移动方向映射到相应的坐标变化。

  5. 初始化战斗网格：

  6. 设置网格大小并创建一个2D数组(grid)，所有格子初始化为null。

  7. 初始化得分和网格：

  8. 遍历creatures数组中的每个生物。

  9. 对于每个生物，在scores对象中将其得分初始化为0。
  10. 使用生物的初始位置和图标将每个生物放置在网格上。

  11. 模拟战斗移动：

  12. 遍历移动次数，从-1开始（表示初始状态）。

  13. 如果是初始状态（move是-1），渲染网格。
  14. 如果是最后一次移动，渲染后退出循环。
  15. 对于每次移动：
    • 根据移动方向确定生物的新位置。
    • 检查新位置是否与另一个生物重叠。
    • 根据重叠或成功的移动更新分数和网格状态。

  16. 渲染网格：

  17. 对于每个网格状态（初始状态和每次移动后）：

    • 显示移动编号或初始状态标识。
    • 打印包含生物或空格子表示的网格状态。
    • 显示所有生物的当前得分。

  18. 返回最终得分：

  19. 在所有移动模拟完成后，返回每个生物的最终得分。

  3. Node.js
  4. Python

  8. Python: 创建一个名称为mythos的文件夹，并添加一个文件名为app.py。

GitHub Copilot建议

使用Copilot提高效率

看看你是否可以使用Copilot来发现代码的复杂性（BigO表示法）。

  1. 打开侧边栏中的GitHub Copilot Chat视图，如果它还没有打开。确保解决方案文件仍然是打开的。
  2. 询问Copilot Chat代码的复杂性。
  3. 询问Copilot Chat使代码更高效。
  4. 再次询问复杂性—它是否更好了？

使用Copilot生成代码注释

  1. 使用Ctrl/Cmd+A选中所有代码。
  2. 按Ctrl/Cmd+I打开内联聊天。
  3. 输入“/doc”
  4. 询问Copilot Chat为函数编写文档。

使用Copilot简化你的代码

  1. 打开侧边栏中的GitHub Copilot Chat。
  2. 输入“/simplify”并按Enter。你还可以在“/simplify”后添加任何文本来给Copilot更多指示。
  3. Copilot Chat建议了什么以简化代码？

有错误吗？

Copilot Chat也可以帮忙！只需复制错误消息并粘贴到Chat中。这通常是Copilot解决问题所需的全部信息。
